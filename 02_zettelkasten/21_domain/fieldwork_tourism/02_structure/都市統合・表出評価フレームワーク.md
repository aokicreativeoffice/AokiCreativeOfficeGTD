# Summary
都市の個性は資源の存在ではなく、「資源の統合」と「入口での表出」によって決まる。本フレームはCore・Latent・Genericの関係と都市UIを評価する。

---

# Core Concept

Urban Identity = Resource × Integration × Interface

---

# 評価軸（2層構造）

## Layer1：Integration（統合）
資源同士が都市構造として結合しているか

## Layer2：Interface（表出/UI）
資源が入口から認知・到達可能か

---

# 1. Integration（統合評価）

## Definition
Core・Latent・Genericが空間・意味・体験として接続されている度合い

---

## Sub Axes

### ① Spatial Integration（空間統合）
- 導線で繋がっているか
- 徒歩・回遊可能か

### ② Narrative Integration（意味統合）
- 一つの物語で説明できるか
- コンセプトが一貫しているか

### ③ Functional Integration（機能統合）
- 滞在・移動・体験が連続するか
- 役割分担が成立しているか

---

## Score（1–5）

| Score | 状態 |
|------|------|
| 1 | 分断（点在） |
| 2 | 局所接続 |
| 3 | 部分統合 |
| 4 | 広域統合 |
| 5 | 完全統合（構造化） |

---

## Diagnostic

- CoreとLatentが繋がっているか？
- GenericがCoreへ導線を持つか？
- 回遊が途切れていないか？

---

# 2. Interface（表出/UI評価）

## Definition
都市の価値が入口で認知され、探索行動に繋がる度合い

---

## Sub Axes

### ① Entry Clarity（入口明確性）
- 駅前で何の街か分かるか

### ② Visual Hook（視覚フック）
- 一目で引っかかる要素があるか

### ③ Cognitive Accessibility（認知容易性）
- 知識なしで理解できるか

### ④ Routing Visibility（導線可視性）
- 次にどこへ行くか分かるか

---

## Score（1–5）

| Score | 状態 |
|------|------|
| 1 | 何も分からない |
| 2 | 情報があれば分かる |
| 3 | 一部認知可能 |
| 4 | 多くが認知可能 |
| 5 | 直感的に理解可能 |

---

## Diagnostic

- 駅前にCoreの気配があるか？
- Latentへの導線が見えるか？
- Genericがノイズ化していないか？

---

# 統合評価モデル

## Total Score

UII Score = Integration × Interface

---

## 解釈

| パターン | 状態 |
|----------|------|
| 高I・高UI | 理想（強い都市） |
| 高I・低UI | 深いが見えない都市 |
| 低I・高UI | 浅いが分かりやすい都市 |
| 低I・低UI | 完全に弱い都市 |

---

# Resource Interaction Model

Generic → Entry（入口）

Core → Anchor（核）

Latent → Depth（深層）

---

## 理想構造

Generic → Core → Latent

---

## 失敗構造

Generic → 終了  
Latent → 孤立  
Core → 埋没  

---

# Failure Patterns

## ① Latent Isolation
潜在資源が接続されていない

## ② Generic Dominance
汎用資源が認知を支配

## ③ Core Weak Signaling
中核資源が入口に出ていない

---

# Application Example

## 名古屋

Integration：2  
- 点在（城・神宮・産業）

Interface：2  
- 駅前で意味不明

→ 「資源があるのに無い都市」

---

## 大宮

Integration：3  
- CoreとLatentが弱接続

Interface：3  
- 一部見えるが弱い

→ 「構造はあるが伝わらない都市」

---

# Design Implications

## 改善レバー

### ① Integration強化
- 導線設計
- 回遊設計
- テーマ統一

---

### ② Interface強化
- 駅前ナラティブ
- 視覚フック
- サイン設計

---

## 戦略

- Genericを入口に使う
- Coreへ誘導する
- Latentで深度を作る

---

# Key Insight

「統合」と「表出」の欠如である
