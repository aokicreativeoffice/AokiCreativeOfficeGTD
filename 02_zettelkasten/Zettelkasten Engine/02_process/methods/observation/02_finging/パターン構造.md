# Pattern Structure

Pattern Structure は、複数の Case（事例）から共通する構造を抽出し、
再利用可能な知識として整理する構造である。

Case は個別事象だが、
Pattern はそれらの背後にある共通構造を表す。

# Pattern の基本構造

パターンは次の要素で構成される。

## 1 現象

何が繰り返し起きているか

例

- 観光都市で滞在時間が長い
- 新規市場で先行企業が有利

---

## 2 共通条件

その現象が起きる条件

例

- 資源密度が高い
- 競争が弱い

---

## 3 構造

現象を生む構造

例

資源密度  
↓  
回遊性  
↓  
滞在時間

---

## 4 例

パターンを示す事例

例

- 京都
- フィレンツェ
- プラハ

---

# Pattern の基本形式

Pattern は次の形式で表される。

条件
↓
構造
↓
結果

---

# Pattern のタイプ

## 相関パターン

A と B が同時に起きる

例

人口増加 ↔ 地価上昇

---

## 因果パターン

A → B

例

価格低下 → 需要増加

---

## 構造パターン

複数要因

A + B + C → D

---

## プロセスパターン

時間変化

A → B → C → D

---

# Pattern 発見の方法

1 Case を集める  
2 共通点を探す  
3 条件を特定する  
4 構造を整理する  

---

# Pattern の品質

良いパターンは次の条件を満たす。

1 再現性  
2 説明力  
3 一般性  
4 予測力  

---

# Pattern の役割

Pattern は

Case
↓
Pattern
↓
Hypothesis

をつなぐ。

複数の事例から共通構造を抽出し、
仮説形成の基礎となる。
