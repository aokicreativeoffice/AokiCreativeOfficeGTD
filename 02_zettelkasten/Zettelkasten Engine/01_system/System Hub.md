# System Hub

## 役割
このVaultにおける「思考の入口」と「制御塔」。
すべての作業はここから開始する。

---

## 基本原則
- 必ず「問い」から入る
- いきなり書かない（まず分類する）
- ノードを辿ってから出力する
- 出力後は必ず検証する

---

## 人力運用フロー（必須）
1. [[Intent Interpretation]]で問いの型を判定
2. [[Mode Selection]]で処理モードを決定
3. [[Task Routing]]で行き先を決める
4. [[Goal Framing]]で目的を固定
5. [[Thinking Protocol]]に従って作業
6. [[Constraint Monitor]]で検証
7. 必要に応じて[[Tool Orchestration]]

---

## 分岐（モード）
- 即答 → [[Direct Answer Mode]]
- 情報不足 → [[Search-Augmented Mode]]
- Vault参照 → [[File-Grounded Mode]]
- 複雑 → [[Stepwise Reasoning Mode]]
- ノート生成 → [[Artifact Generation Mode]]

---

## NG行動
- いきなり結論を書く
- 既存ノートを見ない
- 仮説なしで探索する
