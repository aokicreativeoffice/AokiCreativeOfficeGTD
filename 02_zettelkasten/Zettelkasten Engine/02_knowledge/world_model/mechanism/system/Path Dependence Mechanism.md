# Path Dependence Mechanism

Path Dependence Mechanism（経路依存メカニズム）とは、初期の選択や偶発的な分岐が、その後の制度、技術、行動、資源配分を拘束し、後続の選択肢を狭めていく仕組みである。

---

# 概要

システムは常に最適な状態に向かうわけではない。  
ある時点で採られた選択が、学習、投資、慣習、補完制度、標準化を通じて自己維持的になり、別の道へ移るコストが高くなる。  
そのため「なぜ今これなのか」は、現在の合理性だけでなく、過去の履歴を見る必要がある。

経路依存メカニズムの核心は、

1. 初期分岐
2. 初期選択の定着
3. 補完要素の蓄積
4. 切替コスト上昇
5. 選択肢の拘束

にある。

# メカニズム

## 1. 初期分岐
複数の選択肢が存在する時点で、偶然や局所合理性によって一つが選ばれる。

## 2. 定着
選ばれた方式に沿って設備投資、教育、規格、制度、認知枠組みが蓄積される。

## 3. 補完構造の形成
周辺制度、サプライチェーン、文化、技能、人材配置がその方式に適応する。

## 4. 切替コストの増大
別方式に移行するには再教育、再投資、調整が必要になり、乗換えが困難になる。

## 5. 進路拘束
現在の選択は過去の累積結果に拘束され、代替案があっても採用されにくくなる。

---

# 経路依存の特徴

- 必ずしも最適ではない
- 初期条件の影響が大きい
- 局所合理性が長期拘束を生む
- 補完制度が固定化を支える
- 危機や外部ショックがないと変わりにくい
