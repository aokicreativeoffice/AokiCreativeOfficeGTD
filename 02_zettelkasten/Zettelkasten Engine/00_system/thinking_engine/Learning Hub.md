# Learning Hub

経験・事例・失敗・成功から学び、Vault全体を更新するためのHub。

---

# 学習の基本構造

学習とは、経験を通じて認識・判断・行動の質を高める過程である。

```
Action
↓
Outcome
↓
Reflection
↓
Learning
↓
Update
```

---

# Learning Flow

このVaultにおける学習の流れ

```
Case
↓
Observation
↓
Evaluation
↓
Abstraction
↓
Kernel / Model / Pattern 更新
```

---

# Step 1 Observation（観察）

まず何が起きたかを記録する。

確認すること

- 何をしたか
- 何が起きたか
- 予想と何が違ったか
- 成功か失敗か

参照

[[History_Case_Hub]]

# Step 2 Evaluation（評価）

結果を評価する。

評価軸

- 目標達成度
- コスト
- リスク
- 再現性
- 副作用

[[02_zettelkasten/00_hub/Decision Hub]]
[[Strategy Hub]]

# Step 3 Pattern Recognition（パターン認識）

その結果がどのパターンに当てはまるかを見る。

例

- バブル
- 炎上
- 官僚化
- 独占形成
- 連鎖崩壊

# Step 4 Model Identification（構造認識）

その事例の背後にある構造を特定する。

例

- ロックイン
- 複雑適応系
- 階層
- 情報非対称

[[Model Hub]]

# Step 5 Kernel Extraction（原理抽出）

例

- 制約
- 情報

# Learning Types

## 成功学習

何がうまくいったかを抽出する。

例

- 再現可能な成功条件
- 有効な戦略
- 強いパターン

---

## 失敗学習

何が失敗を生んだかを抽出する。

例

- 判断ミス
- 情報不足
- 構造的制約
- インセンティブ歪み

---

## 反省学習

自分の前提・認知・判断を見直す。

例

- 楽観バイアス
- 過信
- 短期合理
- 見落とし

---

# Learning Questions

学習時に使う問い

- 何が起きたか
- なぜそうなったか
- どの構造が働いたか
- どの原理が支配していたか
- 次回どう変えるか
- 一般化できるか
