# Problem Structure
すべての思考はここから始まる。
Problem Structure は
問題発見
↓
問題定義
↓ 
原因分析 
↓ 
仮説 
↓ 
検証
↓ 
解決策
↓ 
意思決定

という **問題解決の基本プロセス**を扱う。

---

# 全体構造

```mermaid
flowchart TD

A[問題認識]
B[問題定義]
C[原因分析]
D[仮説]
E[検証]
F[解決策]
G[意思決定]
H[実行]
I[評価]

A --> B
B --> C
C --> D
D --> E
E --> F
F --> G
G --> H
H --> I
```

# 基本概念

問題とは、目的 − 現状  =  ギャップのことである。
