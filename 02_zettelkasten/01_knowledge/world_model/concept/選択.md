# 選択

## 定義

複数の可能な状態・行動・戦略の中から  
**一つまたは一部を採用する過程**

を **選択（Selection）** という。

---

# 基本構造

```mermaid
graph TD

Options["複数の選択肢"]
Constraint["制約"]
Information["情報"]
Evaluation["評価"]
Decision["選択"]

Options --> Constraint
Options --> Information

Information --> Evaluation
Evaluation --> Decision
```

選択とは

```
選択肢
+
制約
+
情報
↓
評価
↓
決定
```

である。

---

# 選択の本質

## 1 不確実性の中で起こる

未来は完全には分からないため、  
選択は常に

```
不完全情報
```

のもとで行われる。

---

## 2 制約の中で行われる

選択は

```
可能な行動
```

の中からしか行えない。

つまり制約が

```
選択空間
```

を決める。

---

## 3 結果が差を生む

選択は結果として

```
成功
失敗
```

を生む。

その差が

```
淘汰
適応
```

につながる。

# 選択と探索

探索は

```
可能な選択肢
```

を見つける過程。

選択は

```
その中から決める
```

過程。

つまり

```
探索
↓
選択
```

である。

---

# 選択と競争

競争は

```
複数主体
```

が同じ資源を求める状況。

その結果

```
選択
↓
成功者
```

が生まれる。

---

# 選択と進化

進化は

```
変異
+
選択
+
淘汰
```

によって起こる。

つまり選択は

```
進化の核心
```

である。
