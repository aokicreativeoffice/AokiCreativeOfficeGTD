# 0 思考フロー
```mermaid
flowchart TD

A[Observation<br>観察] --> B[Problem<br>問題定義]

B --> C[Problem Type<br>問題類型]

C --> D[Analysis Framework<br>分析フレーム]

D --> E[Reality Structure<br>現実構造]

E --> F[World Model<br>世界モデル]

F --> G[Kernel<br>普遍原理]

G --> H[Strategy Design<br>戦略設計]

H --> I[Decision Making<br>意思決定]

I --> J[Execution<br>Project / Action]

J --> K[Feedback<br>結果観察]

K --> A
```
# 1 OSの基本構造

Vaultは次の階層で動く。

Purpose / Strategy（hub）  
↓  
Structures（分析OS）  
↓  
World Model（世界の構造）  
↓  
Kernel（普遍原理）  
↓  
Domain / Case  
↓  
Execution

意味

|層|役割|
|---|---|
|Hub|目的・人生戦略|
|Structures|問題解決OS|
|World Model|社会の構造理解|
|Kernel|普遍原理|
|Domain|分野知識|
|Execution|行動|
