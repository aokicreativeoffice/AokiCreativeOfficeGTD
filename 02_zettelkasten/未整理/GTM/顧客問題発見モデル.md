# Problem Discovery（顧客問題発見モデル）

Problem Discoveryとは

**顧客が本当に抱えている問題を発見するプロセス**

である。

多くの事業失敗は  
「問題が存在しない」か  
「問題が弱い」ことに起因する。

事業成功の最初の条件は、強い問題を見つけることである。

---

# Problem Discoveryの基本構造
1. 顧客
2. 背景
3. 症状
4. 真因 
5. 問題

---

# 問題の階層

顧客問題には階層がある。
1. Symptoms（現象） 
2. Operational Problem（業務問題） 
3. Structural Problem（構造問題） 
4. Strategic Problem（経営問題）

営業や事業開発では、上位の問題ほど価値が大きい。

---

# 問題の種類

顧客問題は大きく4種類に分類できる。
1. Revenue Problem（収益） 
2. Cost Problem（コスト） 
3. Risk Problem（リスク） 
4. Growth Problem（成長）

---

# 良い問題の条件

強い問題は次の特徴を持つ。
- 頻繁に発生する
- 深刻である
- 解決コストが高い
- 経営に影響する

弱い問題
- たまにしか起きない
- 重要でない
- 顧客が解決を望んでいない

---

# Problem Discoveryのプロセス
1. 仮説 
2. 顧客観察 
3. ヒアリング 
4. 問題構造分析 
5. 問題定義
---

# 顧客観察

顧客の業務や行動を観察する。

観察対象

- 業務プロセス
- 現場作業
- 情報の流れ
- 意思決定

多くの問題は  
**現場に存在する**。

---

# ヒアリング

顧客に質問する。

例
- 一番困っていることは何ですか
- この業務で時間がかかる部分はどこですか
- 最近増えている問題は何ですか

---

# Root Cause Analysis

問題の原因を分析する。

方法
- [[02_zettelkasten/Zettelkasten Engine/02_process/methods/analysis/なぜなぜ分析]]

例
事故が多い 
↓ 
なぜ 
↓ 
教育不足 
↓ 
なぜ
↓ 
教育制度がない

---

# Problem Definition

最終的に問題を定義する。

良い問題定義
顧客 + 状況 + 問題

例
地方バス会社では 監査対応が複雑で、運行管理者の負担が大きい

---

# Problem DiscoveryとPMF

PMFの構造
Problem × Solution

Problem Discoveryは、PMFの最初のステップである。

---

# Problem DiscoveryとValue Proposition

価値提案は、
1. Problem 
2. Solution 
3. Value
で設計される。

問題が弱いと  
価値提案も弱くなる。

事業成功の順序
1. Problem 
2. Solution
3. Value
4. Sales

営業は極めて困難になる。
