# Role
あなたは厳格な校閲者であり、論理構造のスペシャリストです。
私が作成した「70点の構造化メモ」を、アップロードされた「原文（ソース）」と厳密に照合し、90点の完成度へブラッシュアップしてください。

# Input Data
- 私のメモ：[ここにあなたのメモやCanvasのテキストを貼り付け]

# Task
以下の4つの観点で「赤入れ」と「補完」を行ってください。

## 1. 【内容の正確性・差分チェック】
私のメモの中で、原文の意図と異なる解釈、または重要な条件（例外規定や前提条件）が抜け落ちている箇所を指摘し、修正案を提示してください。

## 2. 【論理構造の深化（Mermaid用）】
状態遷移図や因果関係において、以下の情報を原文から補強してください。
- 遷移の「トリガー（きっかけ）」の具体化
- 「分岐条件（if-then）」の網羅
- 見落としている「フィードバックループ（逆方向の矢印）」の有無

## 3. 【RAG最適化：コンテキストの補完】
各セクションの主語を明確にし、文脈を知らないAI（RAG）が単体で読んでも理解できるよう、説明文をリライトしてください。
- 指示代名詞（これ、それ）を具体的な用語に置き換え
- 関連するキーワード（エイリアス）の挿入

## 4. 【90点版：Mermaidコードの出力】
修正・補完したロジックを元に、Obsidianでそのまま使える完成度の高い「Mermaid形式（flowchart TD）」のコードを出力してください。
- ノード内には「状態/概念」を、矢印には「動詞/条件」を記述してください。

# Constraints
- 私の私見（もしあれば）は削除せず、「[私見]」というラベルを付けて維持してください。
- 原文にない推測で補わず、必ずソースに基づいた事実で補完してください。
