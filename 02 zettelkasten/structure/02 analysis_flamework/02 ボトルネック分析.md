# 概要
Bottleneck Analysis は、システム全体の能力を制限している最も弱い部分（ボトルネック）を特定する分析フレームワークである。
多くのシステムでは、全体能力ではなく最も遅い工程がパフォーマンスを決める。
# 基本原理
もしB が遅い場合、全体速度 = Bになる。
```mermaid
flowchart LR

A[Process A] --> B[Process B]
B --> C[Process C]
C --> D[Process D]

B:::bottleneck

classDef bottleneck fill:#ffcccc
```
# 手順
```mermaid
flowchart TD 
    A[システムの工程を列挙] --> B[各工程の処理能力を測定]
    B --> C[最も制約になっている工程を特定]
    C --> D[改善]
```
# ボトルネック改善の基本サイクル
```mermaid
flowchart TD

A[Identify Constraint]
B[Exploit Constraint]
C[Subordinate Other Processes]
D[Elevate Constraint]
E[Repeat]

A --> B
B --> C
C --> D
D --> E
```
# よくある誤解
多くの改善は、ボトルネック以外を改善してしまう。しかし、ボトルネック以外の改善は全体性能を変えない。
